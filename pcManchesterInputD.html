Input Devices

"

For a computer to be of any use to a person both input and output devices are required. Input devices are used to provide information and data to the computer. The most common input devices would be the mouse and keyboard, but there are dozens of other daily used hardware devices that are input devices such as:

  • Barcode Reader
  • Digital Camera
  • Joystick
  • Graphics Tablet
  • Microphone
  • Webcam and many more

Keyboards contain a processor and circuitry that carries information to and from this processor. There is a grid circuitry, which is known as the key matrix, that sits under each key on the keyboard. Each circuit is broken below every key which acts as a switch, when a key is pressed the circuit is complete allowing a current to flow through. The processor within the keyboard locates closed circuits and compares the location of the circuit on the key matrix to the character map within the read-only memory, or ROM. The character map is basically a lookup table which tells the processor the position of each key in the matrix, and what each keystroke or key combinations represents. An example would be pressing the “t” key by itself would represent the small character t, whereas pressing Shift and t represents the capital letter “T”. This character map can be overwritten using software which can be useful if the user wanted to type in another language using an English keyboard. Keyboards can be either wired or wireless, with the wireless option requiring a receiver to be plugged into the computer and the wired option is typically plugged in via a USB connection.

Both wired and wireless keyboards function in the same way, with the signal from the keyboard being monitored by the computer’s keyboard controller. The keyboard controller processes the input data and forwards it to the operating system or OS. When the OS receives the data it checks to see if the data represents a system level command, for example Ctrl-Alt-Delete. If the input data is not a system level command, the OS sends the data to the currently used application. The application will determine if the data is a command, for example Ctrl-s which would save a word document. Finally, if the data is not a command then the application accepts it as content. If the application decides the data is not content as this stage, the keyboard data is ignored. This entire process takes mere milliseconds.
